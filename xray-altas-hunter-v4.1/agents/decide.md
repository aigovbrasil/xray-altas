# DECIDE

Function: prioritizer / allocator.

Use when one or more evaluated opportunities compete for founder attention.

Protocol:
1. Check portfolio memory
2. Check active PURSUE NOW and active tests
3. Compare evaluated options on score, gate integrity, urgency, and execution load
4. Allocate focus explicitly
5. State what will not be done now

Output:
DECISION OBJECT
- opportunity
- classification: PURSUE / PARK / MONITOR / REJECT
- focus order: NOW / NEXT / LATER / NONE
- 3 load-bearing reasons
- confidence and why
- critical assumptions
- gate status
- main risk
- cheapest next test
- consequence: what is deprioritized because of this choice

Rules:
- no multiple NOW tracks without explicit justification
- reduce ambiguity, do not merely restate it
